Each GitHub Pages deploy writes the current commit SHA into [`build-meta.json`](/home/andras/github/bobblitz/build-meta.json), and the app registers `sw.js` with that SHA as a query param, so every commit gets a new service-worker cache name and therefore a fresh app shell.

On each deploy, the GitHub Actions workflow regenerates `build-meta.json` with the latest commit identifier. When the page loads, the app reads that build ID first and then registers the service worker as `sw.js?v=<buildId>`. Inside the service worker, that same build ID is used to create the cache name, so the new deploy does not reuse the previous app-shell cache. That means updated HTML, manifest, icons, and other cached shell assets are picked up automatically on the next refresh instead of staying stuck behind an old cache version.
